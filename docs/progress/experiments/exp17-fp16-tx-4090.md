# Exp 17: fp16 TX Buffer on RTX 4090

## Hypothesis

The fp16 TX buffer (v15) halves shmem for the TX portion, enabling higher B_SCAT
while maintaining >= 2 blocks/SM. On the A4000, v15 only gave +2.5% because
the register wall prevented higher B_SCAT. On the 4090, the reduced L2 atomic
pressure from higher B should dominate.

## Key Insight

At B=5, shmem = 43.9KB. TX buffer = 2*5*854*4 = 34.2KB (78% of shmem).
With fp16 TX: TX buffer halves to 17.1KB, total shmem at B=9 = 46.5KB.
Two blocks/SM need 93KB < 100KB max. B=9 with 2 blocks/SM is now possible.

Correctness: 2.2e-4 relative error vs fp32 v11 (PASS, within fp16 budget).

## ncu Profile (v15 B=9 ET=4 vs v11 B=5 ET=8)

| Metric | v11 B=5 | v15 B=9 | Change |
|--------|---------|---------|--------|
| Time | 8.5ms | 6.9ms | -19% |
| SM throughput | 48.0% | 61.2% | +13.2pp |
| FMA pipe | 36.5% | 45.9% | +9.4pp |
| L2 atomic pressure | 47.2% | 32.8% | -14.4pp |
| IPC | 1.92 | 2.45 | +28% |
| long_scoreboard | 0.47 | 0.15 | -68% |

B=9 processes 80% more scatterers per batch than B=5, reducing total atomicAdd
operations by ~1.8x. This drops L2 atomic pressure from 47% to 33%, freeing
pipeline bandwidth for compute. IPC jumps 28%.

## Key Findings

1. fp16 TX buffer is the key enabler for higher B_SCAT on 4090
2. B=10 drops to 1 block/SM (shmem=51.6KB, 2x=103.2KB > 100KB)
3. ET=4 is optimal at high B -- fewer registers, avoids spill cliff
4. B=8 ET=8 has 232B spills -> catastrophic -35% performance
5. The 2-block/SM boundary remains critical at ~50KB shmem per block
